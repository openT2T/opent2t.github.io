Getting Started Step 1

Set up your environment (PC or Mac)

[ Improve this document ]

We are using a pretty standard node.js development setup for developing and testing translators. Most of you are probably already familiar with all this, so we will go quickly here.

  • Install Node: https://nodejs.org/en/download/. If prompted, make sure you select the option to install npm, the node package manager.
  • Install Git: http://git-scm.com/downloads. All our source is on github, so this will come in handy.
  • Install your favorite IDE: One good option is Visual Studio Code. It is really light weight, and available for PC or Mac.

Great. Now let's clone some repos so we can look at code. You will find all our repos on GitHub at http://github.com/openT2T. We recommend that you put all these repos in the same directory.

$ mkdir openT2T
$ cd openT2T
$ git clone https://github.com/openT2T/translators.git
$ git clone https://github.com/openT2T/onboarding.git
$ git clone https://github.com/openT2T/voice.git
$ git clone https://github.com/openT2T/console.git

You may notice that there are other repos under http://github.com/openT2T. We will get to those later.

To play with translators during development on Windows, we will use the IoT explorer for AllJoyn. This is available in the Windows store as a free download here: https://www.microsoft.com/store/apps/9nblggh6gpxl. For other platforms, we have some sample apps available in the sampleapps repo.

In order for IoT explorer for AllJoyn to work with devices running on your local Windows machine, you need to set a loopback exception. Here's how you do that:

  • Download the Loopback Exemption Manager from http://loopback.codeplex.com/ and run it.
  • Find the Alljoyn explorer app and give it a loopback exception.

Great. Now you're all set up to write some translators.
